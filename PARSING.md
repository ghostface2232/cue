# Cue 파서 테스트 케이스

이 문서는 한국어 자연어 파서(IDateParser) 검증용 테스트 케이스 모음입니다. quick-add 한 줄 입력을 날짜(ScheduledWhen)·반복(RecurrenceRule)과 순수 제목으로 분리하는 동작을 확인합니다.

## 표기 규칙

- 할 일의 날짜는 When 하나뿐입니다(별도의 마감일 슬롯 없음). When = ScheduledWhen, 두 변형 중 하나입니다. **OnDate**(구체 날짜) 또는 **Unscheduled**(날짜 없음 = "언젠가").
- "까지·마감·N일 안에" 같은 기한 표현도 인식·제거하지만, 별도 마감일이 아니라 When(OnDate)으로 들어갑니다.
- "언젠가·나중에·…" 같은 마커도 인식·제거하지만, 별도 Someday 상태가 아니라 Unscheduled로 둡니다(= "언젠가" 목록).
- Recurrence = RRULE 문자열 요약입니다.
- 상대 표현(오늘·내일·모레·다음 주 등)은 파싱 시점 사용자 타임존 기준 절대 날짜로 박힙니다. 따라서 아래 When 값의 괄호 안 표기는 의미이며, 실제 저장값은 실행 시점마다 달라지는 구체 날짜입니다.
- 빈 칸은 - 로 표기합니다.

## 3. 시각 포함

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 내일 오후 3시 회의 준비 | 회의 준비 | OnDate(내일 15:00) | - |
| 오늘 저녁 7시 저녁 약속 | 저녁 약속 | OnDate(오늘 19:00) | - |
| 모레 오전 10시 화상 인터뷰 | 화상 인터뷰 | OnDate(모레 10:00) | - |
| 새벽 6시 공항 픽업 나가기 | 공항 픽업 나가기 | OnDate(다가오는 06:00) | - |
| 다음 주 수요일 오후 2시 30분 거래처 미팅 | 거래처 미팅 | OnDate(다음 주 수요일 14:30) | - |
| 점심때 처방약 먹기 | 처방약 먹기 | OnDate(오늘 12:00경) | - |
| 한 시간 후에 빨래 건조기 돌리기 | 빨래 건조기 돌리기 | OnDate(현재 +1시간) | - |
| 오후 4시 반 아이 하원 픽업 | 아이 하원 픽업 | OnDate(오늘 16:30) | - |
| 오늘 3시 미팅 | 미팅 | OnDate(오늘 15:00) | - |
| 내일 3시 미팅 | 미팅 | OnDate(내일 15:00) | - |
| 내일 9시 회의 | 회의 | OnDate(내일 09:00) | - |

두 번째(저녁) 행은 까다로운 케이스입니다. 앞의 오늘 저녁 7시만 시각으로 잡고, 제목의 저녁 약속에 남은 저녁은 시각으로 다시 잡으면 안 됩니다. 저녁/밤 같은 시각대 단어는 구체 시각으로 변환합니다(4절 참고).

오전·오후 표기가 없는 시각은 모호합니다. ToDo 특성상(직장인이 새벽 일정을 잡는 일은 드물고, 그럴 땐 "새벽 3시"·"오전 3시"처럼 명시함) 다음과 같이 보정합니다.

- 1~6시: 날짜와 무관하게 오후로 봅니다(13~18시). 그래서 오늘 3시·내일 3시 모두 15:00입니다.
- 7~11시: 아침 일정(예: 9시 회의)일 수 있어 기본은 오전입니다. 단, 오늘 날짜이고 그 오전 시각이 이미 지났으면 오후로 봅니다(오전 10시에 입력한 오늘 9시 → 21:00).
- 0·12·13~23시, 그리고 오전/오후/새벽/저녁을 명시한 경우는 그대로 둡니다.

날짜·요일이 전혀 없는 시각만 입력한 경우(3시·오후에·점심 먹고 등)는 기본이 오늘이되, 위 보정을 거친 시각이 이미 현재보다 이전이면 내일로 넘깁니다. 예) 16시에 입력한 3시 → 내일 15:00, 23시에 입력한 9시(→21:00) → 내일 21:00. 날짜·요일을 명시한 입력(오늘 3시·금요일 3시)은 사용자가 날짜를 고정한 것이므로 지나간 시각이어도 그 날짜에 그대로 둡니다.

## 4. 시각대 단어 → 대표 시각

시각이 따로 없는 추상 시각대 단어는 각자의 대표 시각으로 변환합니다.

| 단어 | 대표 시각 |
|---|---|
| 새벽 | 06:00 |
| 아침 | 08:00 |
| 오전 | 10:00 |
| 점심 | 12:00 |
| 오후 | 15:00 |
| 저녁 | 18:00 |
| 밤 | 21:00 |

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 오늘 저녁 운동 | 운동 | OnDate(오늘 18:00) | - |
| 저녁에 강아지 산책시키기 | 강아지 산책시키기 | OnDate(오늘 18:00) | - |
| 밤 산책 나가기 | 산책 나가기 | OnDate(오늘 21:00) | - |
| 오후에 미팅 | 미팅 | OnDate(오늘 15:00) | - |

시각대 단어에 구체 시각이 붙으면(저녁 7시, 밤 9시, 새벽 6시) 그 시각이 우선하며 단어는 오전/오후만 결정합니다 → 19:00, 21:00, 06:00.

## 5. 기한 표현(까지·마감·N일 안에) → When

할 일의 날짜는 하나뿐이므로, 기한 표현은 별도 마감일이 아니라 When(OnDate)으로 들어갑니다. 표현 자체는 인식·제거됩니다.

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 금요일까지 보고서 제출 | 보고서 제출 | OnDate(다가오는 금요일) | - |
| 이번 달 말까지 카드값 정산 | 카드값 정산 | OnDate(이번 달 말일) | - |
| 내일까지 과제 마무리 | 과제 마무리 | OnDate(내일) | - |
| 다음 주 수요일까지 견적서 회신 | 견적서 회신 | OnDate(다음 주 수요일) | - |
| 25일까지 관리비 납부 | 관리비 납부 | OnDate(다가오는 25일) | - |
| 월요일 오전까지 발표 자료 마감 | 발표 자료 마감 | OnDate(다가오는 월요일 오전) | - |
| 3일 안에 교환 신청 넣기 | 교환 신청 넣기 | OnDate(+3일) | - |
| 5월 31일까지 종합소득세 신고 | 종합소득세 신고 | OnDate(5월 31일) | - |

## 6. 반복

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 매주 월요일 운동 | 운동 | - | FREQ=WEEKLY;BYDAY=MO |
| 매일 아침 영양제 챙겨 먹기 | 영양제 챙겨 먹기 | - | FREQ=DAILY (아침) |
| 매월 1일 가계부 정산 | 가계부 정산 | - | FREQ=MONTHLY;BYMONTHDAY=1 |
| 매주 금요일 주간 회고 작성 | 주간 회고 작성 | - | FREQ=WEEKLY;BYDAY=FR |
| 격주 수요일 스터디 모임 | 스터디 모임 | - | FREQ=WEEKLY;INTERVAL=2;BYDAY=WE |
| 평일 아침 7시 기상 | 기상 | 시각 07:00 | FREQ=WEEKLY;BYDAY=MO,TU,WE,TH,FR |
| 매년 어머니 생신 챙기기 | 어머니 생신 챙기기 | - | FREQ=YEARLY |
| 매주 일요일 화분에 물 주기 | 화분에 물 주기 | - | FREQ=WEEKLY;BYDAY=SU |
| 30분마다 일어나서 스트레칭 | 일어나서 스트레칭 | - | FREQ=MINUTELY;INTERVAL=30 |

마지막 행의 분 단위 반복은 보강 레이어 룰(N분마다 → FREQ=MINUTELY;INTERVAL=N)로 처리됩니다.

## 8. "언젠가" 마커 → Unscheduled

별도 Someday 상태는 없습니다. 아래 마커는 인식·제거되지만 결과 When은 Unscheduled이고, 이런 할 일은 "언젠가" 목록에 모입니다.

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 언젠가 제주도 한 달 살기 | 제주도 한 달 살기 | Unscheduled | - |
| 나중에 기타 배우기 | 기타 배우기 | Unscheduled | - |
| 다음에 만나서 회포 풀기 | 만나서 회포 풀기 | Unscheduled | - |
| 담에 영화 보기 | 영화 보기 | Unscheduled | - |
| 시간 날 때 옛날 사진 백업하기 | 옛날 사진 백업하기 | Unscheduled | - |
| 여유되면 자전거 정비하기 | 자전거 정비하기 | Unscheduled | - |
| 기회 되면 부모님과 여행 | 부모님과 여행 | Unscheduled | - |

"언젠가" 마커: 언젠가·나중에·다음에·담에·시간 나면·시간 날 때·여유되면·여유 생기면·기회 되면 등(띄어쓰기 유무 무관). 또한 퀵애드에서 날짜·시각 표현이 전혀 없는 할 일은, Today 목록에서 추가할 때만 오늘로 배치되고 그 외 목록(Upcoming 포함)에서는 Unscheduled로 남아 "언젠가"에 들어갑니다.

날짜 표현이 문장 중간이나 뒤에 와도 결과는 동일해야 합니다.

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 보고서 금요일까지 끝내기 | 보고서 끝내기 | OnDate(다가오는 금요일) | - |
| 회의 준비 내일 오후 3시 | 회의 준비 | OnDate(내일 15:00) | - |

## 10. 오인식 방지

날짜·시각처럼 보이는 토막이 제목의 일부인 경우입니다. 모두 When=Unscheduled, 제목=입력 전체가 정답입니다.

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 3월의 라이온 만화책 사기 | 3월의 라이온 만화책 사기 | Unscheduled | - |
| 내일로 기차표 예매하기 | 내일로 기차표 예매하기 | Unscheduled | - |
| 오늘의집 앱에서 소파 주문 | 오늘의집 앱에서 소파 주문 | Unscheduled | - |
| 24시 김밥집에 도시락 미리 주문 | 24시 김밥집에 도시락 미리 주문 | Unscheduled | - |
| 천 원짜리 우표 사 두기 | 천 원짜리 우표 사 두기 | Unscheduled | - |
| 일요일의 데이트 플레이리스트 만들기 | 일요일의 데이트 플레이리스트 만들기 | Unscheduled | - |

여기서 3월·내일·오늘·24시·천(숫자)·일요일을 날짜나 시각으로 잡으면 실패입니다. 라이브러리 단독으로는 오인식 가능성이 있어 보강 규칙 검증 지점입니다.

## 11. 복합

날짜·시각·반복이 한 문장에 섞인 경우입니다. 할 일의 날짜는 하나뿐이므로, 날짜+시각대는 하나의 When으로 합쳐지고 두 표현 모두 제목에서 제거됩니다.

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 내일 오전에 자료 모아서 기획안 제출 | 자료 모아서 기획안 제출 | OnDate(내일 오전) | - |
| 오늘 저녁 7시 약속 끝나고 마트 들르기 | 약속 끝나고 마트 들르기 | OnDate(오늘 19:00) | - |
| 다음 주 월요일부터 매일 30분씩 영어 공부 | 영어 공부 | OnDate(다음 주 월요일, 앵커) | FREQ=DAILY |

한 입력에 서로 다른 두 날짜 표현(예: "내일"과 "금요일까지")이 동시에 오면, 먼저 인식되는 룰이 단 하나의 When을 차지합니다(기한 표현 룰이 일반 날짜 룰보다 앞).

지원하는 축약·구어 요소:

- 내일=낼, 내일모레=낼모레.
- 네이티브 시각: 한~열두시(한시·세시·네시·…·열두시). 시 단위만 네이티브를 받고, 분은 숫자입니다.
- 다음 주 = 다음/담주. 요일과 붙으면(다음 금요일·담주 목요일) 모두 다음 주 그 요일로 봅니다.
- 날짜·시각에 붙은 조사(까지·에·에는·은·는·도·부터·쯤 등)는 의미를 유지한 채 제목에서 분리합니다. '오늘의집'·'내일로'처럼 다른 단어에 붙은 경우는 여전히 날짜로 오인하지 않습니다.

## 13. 다음·담 분기와 요일 줄임

`다음`/`담`은 뒤에 오는 토큰으로 날짜와 무날짜를 구분합니다. 뒤 토큰이 요일·날짜이면 날짜로, `다음에`/`담에`처럼 `에`가 직접 붙거나 막연한 동작이 오면 "언젠가" 마커로 봅니다.

| 입력 | 제목 | When | Recurrence |

## 15. 구어 반복과 식사 시점

반복 표현은 의미가 같은 구어 변형도 같은 RRULE로 정규화합니다.

| 입력 | 제목 | When | Recurrence |
|---|---|---|---|
| 해마다 엄마 생신 챙기기 | 엄마 생신 챙기기 | - | FREQ=YEARLY |
| 30분에 한 번씩 스트레칭하기 | 스트레칭하기 | - | FREQ=MINUTELY;INTERVAL=30 |
| 매주 월욜마다 운동 가기 | 운동 가기 | - | FREQ=WEEKLY;BYDAY=MO |
| 평일엔 7시에 일어나기 | 일어나기 | 시각 07:00 | FREQ=WEEKLY;BYDAY=MO,TU,WE,TH,FR |

| 금욜 저녁마다 가계부 정리 | 가계부 정리 | - | FREQ=WEEKLY;BYDAY=FR |
| 평일마다 스트레칭 | 스트레칭 | - | FREQ=WEEKLY;BYDAY=MO,TU,WE,TH,FR |

## 보강 대상 메모

대표 표현 대부분은 이미 보강 레이어 룰로 처리되고 회귀 테스트로 고정돼 있습니다. 처리되는 항목: 매일·매주(요일)·매월(N일)·매년, 격주, 평일(평일마다·매주 평일), 매주 없는 요일마다(목욜마다·금욜 저녁마다), 30분마다(N분마다), 해마다, N분에 한 번씩; 점심·점심때·점심 먹고(→정오 12시)·새벽·아침·오후·저녁 같은 시각대 표현; 다음·담 날짜 분기, 요일 줄임(욜), 이따, 글피, 이번 달 말일; 제목 속 날짜형 단어 오인식 방지(내일로·오늘의집·3월의·24시 등).

라이브러리·보강 레이어 모두에서 아직 공백인 항목만 남깁니다.

- 아침저녁(하루 두 번)처럼 한 작업에 복수 시점을 거는 반복 표현 — 단일 RRULE로 깔끔히 표현하기 어려움
- 월급날처럼 의미 기반으로 날짜를 가리키는 표현 — 별도 사전/규칙 필요 (말일은 "이번 달 말일/말"로 지원, 월급날은 동반된 "25일" 등이 있어야 날짜가 잡힘)

실패한 입력은 이 표에 행을 추가하고, 보강 규칙을 붙인 뒤 같은 행으로 회귀 검증하면 됩니다.
